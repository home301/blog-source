# OpenClaw 실사용 팁 — AI 비서를 제대로 부리는 법

> 설치는 끝났다. 이제 제대로 쓰는 방법을 정리한다.

<!-- more -->

---

## 규칙을 정해줘라

AI 에이전트는 자유도가 높은 만큼 의도치 않은 행동을 한다. "알아서 잘 해줘"는 통하지 않는다.

워크스페이스에 RULES.md를 만들어서 규칙을 명시하면 훨씬 안정적이다.

실제로 운영하면서 정한 규칙들:

### 계획은 승인 후 실행

에이전트에게 작업을 시키면 먼저 계획을 세우게 한다. 그 계획을 확인하고 승인한 뒤에 실행. 승인된 계획을 에이전트가 임의로 바꾸지 못하게 한다.

이게 없으면 에이전트가 중간에 "더 좋은 방법이 있을 것 같아서" 하고 마음대로 방향을 튼다.

### 결과는 반드시 보고

간단하지만 중요한 규칙이다. 작업이 끝나면 됐든 안 됐든 결과를 말하게 한다.

이걸 안 정해두면 에러가 났는데 조용히 넘어가거나, 성공했는데 아무 말 없이 다음 작업으로 넘어가는 경우가 있다.

### GPU 프로그램은 동시에 하나만

Ollama(LLM), Stable Diffusion Forge, ComfyUI 같은 GPU 프로그램을 동시에 실행하면 VRAM이 부족해져서 드라이버가 죽을 수 있다. 하나를 끄고 다른 하나를 켜는 규칙을 정해두면 된다.

---

## GPU 크래시에 대비하라

VRAM 한계를 넘기면 CUDA 크래시가 발생한다. 이 상태에서는 `nvidia-smi`조차 응답하지 않는다.

문제는 **소프트 리부트로 복구되지 않는다**는 점이다. 전원을 완전히 끄고 다시 켜는 콜드 부트가 필요하다.

모니터 앞에 앉아있으면 전원 버튼을 누르면 되지만, 원격 접속 중이라면? RTC wake를 쓴다:

```bash
# 전원 완전 차단 후 60초 뒤 자동 켜짐
sudo rtcwake -m off -s 60
```

에이전트에게 이 명령을 가르쳐두면 GPU 크래시 감지 시 자동으로 콜드 부트를 실행하고, 복귀 후 상태를 보고하게 할 수 있다.

예방이 최선이다:

- VRAM 제한 설정 (예: `OLLAMA_MAX_VRAM=14G`)
- 16GB VRAM을 넘는 모델은 올리지 않기
- GPU 프로그램 동시 실행 금지

## 성격과 말투를 잡아줘라

기본 상태의 에이전트는 대화마다 톤이 달라질 수 있다. SOUL.md에서 성격을 설정하면 일관된 대화가 가능하다.

정해두면 좋은 것들:

- **이름** — 에이전트를 뭐라고 부를지. 이름이 있으면 대화가 자연스러워진다
- **말투** — 높임말/반말, 간결한지 설명이 긴지
- **금지 사항** — 불필요한 이모지, 과도한 인사말, "좋은 질문이에요!" 같은 사족

USER.md에는 사용자 쪽 설정을 넣는다:

- 선호하는 언어
- 높임말 사용 여부
- 이름 호출 여부

한번 설정하면 세션이 바뀌어도 유지된다. 에이전트가 매번 이 파일을 읽고 시작하기 때문이다.

---

## 기억은 파일로 남겨라

에이전트는 세션마다 기억이 초기화된다. "이거 기억해"라고 말해도 파일에 쓰지 않으면 다음 세션에서 모른다.

| 파일 | 용도 |
|------|------|
| MEMORY.md | 장기 기억 — 중요한 결정, 교훈, 선호도 |
| memory/YYYY-MM-DD.md | 일별 기록 — 그날 한 일의 로그 |
| USER.md | 사용자 정보 — 말투, 스타일 선호 |
| RULES.md | 운영 규칙 |

"기억해둬"라고 하면 에이전트가 적절한 파일에 기록하고, 다음 세션에서 자동으로 읽는다.

반대로, 파일에 없는 건 기억하지 못한다. 중요한 건 반드시 파일로.

---

## 하트비트를 활용하라

하트비트는 에이전트가 주기적으로 깨어나서 할 일을 체크하는 기능이다. HEARTBEAT.md에 체크리스트를 적어두면 된다.

활용 예:

- 이메일 확인
- 캘린더 다가오는 일정 알림
- 특정 서비스 상태 모니터링
- 날씨 체크

너무 자주 깨우면 API 비용이 올라가고, 너무 드물면 의미가 없다. 30분 간격 정도가 적당하다.

## 워크스페이스를 정리하라

`~/.openclaw/workspace/`가 에이전트의 작업 공간이다. 여기에 파일이 쌓이면 에이전트가 컨텍스트를 파악하는 데 방해가 된다.

권장 구조:

```
workspace/
├── SOUL.md          # 에이전트 성격
├── USER.md          # 사용자 정보
├── MEMORY.md        # 장기 기억
├── RULES.md         # 운영 규칙
├── HEARTBEAT.md     # 하트비트 체크리스트
├── IDENTITY.md      # 이름, 닉네임
└── memory/          # 일별 기록
    ├── 2026-03-05.md
    ├── 2026-03-06.md
    └── ...
```

프로젝트별 파일은 하위 디렉토리로 분리하는 게 깔끔하다.

## 정리

OpenClaw 에이전트는 도구다. 도구는 쓰는 사람이 어떻게 세팅하느냐에 따라 결과가 달라진다. 핵심은:

1. **규칙을 명확하게** — 암묵적 기대 대신 명시적 규칙
2. **기억을 파일로** — 말로 한 건 사라진다
3. **실패에 대비** — GPU 크래시, API 장애 등의 복구 절차를 미리 정해두기
4. **성격을 잡아주기** — 일관된 대화 경험
